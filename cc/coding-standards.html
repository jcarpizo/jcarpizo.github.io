Basic Coding Standard¶

1. Files [PSR-1]¶

1.1. PHP Tags¶

PHP code MUST use the long <?php ?> tags or the short-echo <?= ?> tags; it MUST NOT use the other tag variations.

1.2. Character Encoding¶

PHP code MUST use only UTF-8 without BOM.

1.3. Side Effects¶

A file SHOULD declare new symbols (classes, functions, constants, etc.) and cause no other side effects, or it SHOULD execute logic with side effects, but SHOULD NOT do both.

The phrase "side effects" means execution of logic not directly related to declaring classes, functions, constants, etc., merely from including the file.

"Side effects" include but are not limited to: generating output, explicit use of require or include, connecting to external services, modifying ini settings, emitting errors or exceptions, modifying global or static variables, reading from or writing to a file, and so on.

The following is an example of a file with both declarations and side effects; i.e, an example of what to avoid:

<?php
// side effect: change ini settings
ini_set('error_reporting', E_ALL);

// side effect: loads a file
include "file.php";

// side effect: generates output
echo "<html>\n";

// declaration
function foo(){
    // function body
}

The following example is of a file that contains declarations without side effects; i.e., an example of what to emulate:

<?php
// declaration
function foo(){
    // function body
}

// conditional declaration is *not* a side effect
if (! function_exists('bar')) {
    function bar(){
        // function body
    }
}

2. Namespaces¶

  • The fully qualified class name MAY have one or more sub-namespace names
  • Underscores are not allowed
  • Only Alphabetic characters are allowed
  • Namespaces are written in all lower case
  • All class names MUST be referenced in a case-sensitive fashion [PSR-4].

3. Classes¶

Namespaces and classes MUST follow an "autoloading" PSR: [PSR-0] , [PSR-4]. This means each class is in a file by itself, and is in a namespace of at least one level: a top-level vendor name.

Class names MUST be declared in StudlyCaps.

Multiple class declarations MUST NOT be in one file. Only one class declaration per file is allowed.

Code written for PHP 5.3 and after MUST use formal namespaces.

For example:

<?php
// PHP 5.3 and later:
namespace vendor\model;

class Foo{
}

File location:

For PSR-0:

Fully qualified class name File location
app\model\Foo app/model/Foo.php

For PSR-4:

Fully qualified class name Vendor base directory File location
foovendor\app\model\Foo vendor/foo vendor/foo/app/model/Foo.php

3.1. Constants¶

Class constants MUST be declared in all upper case with underscore separators [PSR-1].

For example:

<?php
namespace vendor\model;

class Foo{
    const VERSION = '1.0';
    const DATE_APPROVED = '2012-06-01';
}

3.2. Properties¶

  • MUST NOT contain underscores.
  • MUST NOT be declared in StudlyCaps.
  • MUST be declared in camelCase.

3.3. Methods¶

  • MUST be declared in camelCase().
  • Type declarations (Type Hinting) MUST be used
    • Class/interface name
    • array
  • Type declarations (Type Hinting) MUST NOT be used for scalar/basic typehints
    • int, boolean, string e.g.

4. Interfaces¶

Interfaces MUST be suffixed by Interface: e.g.

<?php
namespace vendor\model;

interface FooInterface{
}

5. Abstract classes¶

Abstract classes MUST be prefixed by Abstract: e.g.

<?php
namespace vendor\model;

abstract class AbstractFoo{
}

6. Traits¶

Traits MUST be suffixed by Trait: e.g.

<?php
namespace vendor\model;

trait FooTrait{
}
