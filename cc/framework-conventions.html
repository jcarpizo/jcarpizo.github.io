FW-Basic-Conventions¶

1. File inclusion¶

The PHP-Functions include and require MUST NOT be used to include PHP files.

Note

Require statments are allowed in the application bootstrap file (only there), to load other configuration files.

2. Static-Methods¶

Are NOT RECOMMENDED.

3. Password hashing¶

For PHP version 5.5 and above the internal Password Hashing Functions MUST be used.

password_hash password_verify

Example:

$hashedPassword = password_hash("pwd" ,PASSWORD_BCRYPT, ['cost' => 12]);

Verification:

if (password_verify("pwd" , hashedPassword) === false) {
    throw new Exception("Wrong password");
}

Note

PHP versions below 5.5 MUST use the "ircmaxell/password-compat" library

4. Random string¶

Cryptographically strong pseudo-random strings SHOULD be generated with the the library "ircmaxell/random-lib"

5. User-defined functions¶

SHOULD NOT be defined. Only methods of classes SHOULD be used

6. Date¶

PHP provides the DateTime class. This SHOULD be used in any case where a Date or Time is used.

Danger

Date functions MUST NOT be used!

Warning

The default time zone MUST be set in the bootstrap file to UTC!
date_default_timezone_set("UTC");
ini_set('default_timezone', 'UTC');
OR in the php.ini

Note

DateTime-Class documentation: http://php.net/manual/en/class.datetime.php

Date functions blacklist:

checkdate
date_*
date
getdate
gettimeofday
gmdate
gmmktime
gmstrftime
idate
localtime
microtime
mktime
strftime
strptime
strtotime
time
timezone_abbreviations_list
timezone_identifiers_list
timezone_location_get
timezone_name_from_abbr
timezone_name_get
timezone_offset_get
timezone_open
timezone_transitions_get
timezone_version_get

7. Superglobals¶

MUST NOT be used in any context of the application! All "globals" should be retrieved from the Request object

Symfony\Component\HttpFoundation\Request

Example:

<?php

// $_GET
$request->query->get('name');

// $_POST
$request->request->get('name');

// $_SERVER
$request->server->get('name');

// $_COOKIES
$request->cookies->get('name');

// $_FILES
$request->files->get('name');

// headers
$request->headers->get('name');

// $_SESSION
$request->getSession()->get('name');

8. Script termination¶

Script termination functions like die or exit MUST NOT be used! They MAY be used only for debugging! Exceptions MUST be used instead of termination functions!

9. Comparison Operators¶

For boolean and null values the Identical operator must be used!

Warning

The is_null function MUST NOT be used!

For example:

if ($foo === true) {
    // true
}

if ($foo === false) {
    // false
}

if ($foo !== true) {
    // false
}

if ($foo === null) {
    // null
}

if ($foo !== null) {
    // not null
}

10. Exceptions¶

  • Generic exceptions must neither be thrown nor be caught!
  • Empty catch blocks are not allowed! At least a comment must be provided, which explains why the block is empty!

11. Class instantiation¶

In general all classes SHOULD be passed as a parameter. In some cases a manual instantiation is allowed:

Context Allowed classes Description
\ \ All native PHP-Classes in all contexts
app\service\ hsapiclient\ Models of the HS API-Client
app\service\ app\model\ Domain models
app\service\ Doctrine\Common\Collections\ Doctrine Collections
app\controller\ Symfony\Component\HttpFoundation\[ Response,RedirectResponse,JsonResponse] Response objects
