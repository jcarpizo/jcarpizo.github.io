PHP-Doc-Comments¶

Note

The full Documentation for the PHP-Doc-Comment can be found here: https://github.com/phpDocumentor/fig-standards/tree/master/proposed

General¶

  • Doc-Comments MUST NOT have a blank line at the end of the deceleration
  • Inline Doc-Comments MUST NOT be used. Use instead: /**/
  • Comments of parameters, return statements and exceptions MUST start with a capital letter and end with a full stop.

1. Classes¶

  • A class SHOULD contain a short description

Example:

<?php

namespace foo\bar;

/**
 * Short description of the class
 */
class Foo{


}

2. Properties¶

All properties MUST have a Doc-Comment with the @var Tag, that specifies the Data types of the property

Example:

<?php

namespace foo\bar;

/**
 * Short description of the class
 */
class Foo{

    /**
     * @var int
     */
    private $foo;

    /**
     * @var float
     */
    private $bar;

    /**
     * @var \foo\bar\FooBar
     */
    private $fooBar;

}

3. Methods¶

  • All methods SHOULD contain a short description
  • If a method uses parameters, then the @param Tag MUST be provided for each parameter with:
    • The data type
    • A comment that describes the parameter
  • If a method returns data, then a @return Tag MUST be provided with:
    • The data type
    • A comment that describes the returned result
    • Multiple returned data types MUST be concatenated with a pipe (|)
    • Void MUST NOT be used as return data type
  • If a method throws Exceptions or uses classes that throws Exceptions, then a @throws Tag MUST be provided with:
    • The type of the Exception
    • A comment that describes when or why the Exception occurs

Example:

<?php

namespace foo\bar;

/**
 * Short description of the class
 */
class Foo{

    /**
     * Short description
     *
     * @param $bar int Describe bar here.
     * @param $fooBar \foo\bar\FooBar Describe FooBar here.
     */
    public function foo($bar, $fooBar){
        // ...
    }

    /**
     * Short description
     *
     * @param $foo \foo\bar\Foo|null Parameter description.
     * @throws \Exception Describe why.
     * @throws \ErrorException Describe why.
     * @return string Output description.
     */
    public function bar($foo = null){
        if ($foo === null) {
            throw new Exception("msg");
        }
        // throws ErrorException
        $this->service->validate();
        return "ok";
    }

    /**
     * Short description
     *
     * @param $foo \foo\bar\Foo|null Parameter description.
     * @return string|int Output description.
     */
    public function fooBar($foo = null){
        if ($foo === null) {
            return 1;
        }
        return $foo->__toString();
    }
}

4. Local variables¶

Missing type declarations (Type Hinting) MUST be declared for local variables with a multi line comment.

Example:

<?php

// PHP view file.

/* @var $someVar string */
echo $someVar;

Data types¶

  • string, the element to which this type applies is a string of binary characters.
  • int, the element to which this type applies is a whole number or integer.
  • bool, the element to which this type applies only has state TRUE or FALSE.
  • float, the element to which this type applies is a continuous, or real, number.
  • object, the element to which this type applies is the instance of an undetermined class.
  • mixed, the element to which this type applies can be of any type as specified here. It is not known on compile time which type will be used.
  • array, the element to which this type applies is an array of values.
  • resource, the element to which this type applies is a resource per the definition of PHP.
  • void, this type is commonly only used when defining the return type of a method or function.
