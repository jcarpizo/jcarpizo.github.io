Feil, Feil & Feil PHP-Coding-Conventions¶

Note

You should really take the time and read the whole Coding Conventions. Coding standards are an important tool to for software quality improvement! So please follow these guidelines and help us to make software maintenance easier.

TL;DR

Project-Setup¶

Project-Setup

Code formatting¶

We use the PSR-1 AND PSR-2 standard with following modifications:

  • The opening curly bracket of methods stays in the same line without a whitespace.
  • Abstract classes has the prefix "Abstract" e.g. AbstractFoo
  • Interfaces has the postfix "Interface" e.g. FooInterface
  • Traits has the prefix Trait e.g. TraitFoo

PHP-Doc-Comments¶

Please provide at least following comments for classes,interfaces or traits:

  • A short description for the purpose
  • Please don't use the @author tag.

For Class Members:

  • The @var tag with the data type of the member (string,integer,boolean,float, array, YourObject, YourListOfObjects[])

For Methods:

  • If parameters are used then foreach parameters an @param tag with the data type
  • If a return value will be produced, then a @return tag with the data type(s) e.g. @return integer|null
  • If the method throws or produce an exception, then a @throws tag with the exception types e.g. fooFooException|barBarException

Comments¶

  • One Space after comment open tag is REQUIRED
  • MUST start with a capital letter
  • MUST end with a full stop
  • Perl-style comments (#) MUST not be used
  • PHP-Doc-Comments MUST not be used

Routing¶

  • Resource names MUST be in plural e.g. customers
  • Resource names SHOULD be all lower case
  • A Route MUST begin with a slash and end with one (in views without an leading slash!)
  • Use HTTP-Verbs

Dependencies¶

  • All dependencies MUST be managed by composer.
  • Dependencies MUST NOT be installed manually!
  • DEV/Master-Branches versions MUST NOT be used
  • The Caret (^) operator SHOULD be used for versions

Front-End-Dependencies¶

Use bower-installer for all Front-End-Dependencies. Don't commit them to the SVN!

FW-Basic-Conventions¶

  • No Superglobals !!!
  • No Script termination commands (exit, die)
  • No file inclusion
  • No static methods
  • No user defined function! Use Classes and Methods!
  • Use PHP-Internal password_* functions for password hashing
  • Cryptographically strong pseudo-random strings are recommended (don't use mt_rand)
  • Use always the DateTime class of PHP instead of date_* functions
  • Use the type safe Comparison Operators (===) for boolean and null data types
  • Generic exceptions must neither be thrown nor be caught! Empty catch blocks are not allowed! At least a comment must be provided, which explains why the block is empty!
  • In general all classes SHOULD be passed as a parameter. No manual instantiation!

FW-Controllers¶

  • Use max. 4 Services
  • User input should be handled with the serialization library (paste the whole object to your method instead of dozens of parameters, or instead of a dozens of setter calls)
  • No Validation in a controller! This task belongs into a Service!

FW-Services¶

  • Must always implement an interface
  • All other Classes must use only this interface! Never the concrete implementation!
  • Can and should use other services

Database¶

Naming:

Type Naming
Entity Singular, lower case, underscore for spacing
Attribute All lower case, underscore for spacing
Relationship Entity Combination of both singular entity names, spaced with an underscore (e.g. account_address)
