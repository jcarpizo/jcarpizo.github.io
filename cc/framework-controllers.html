FW-Controllers¶

1. Services¶

  • A controller SHOULD NOT use more than 4 Services
  • A controller MUST NOT instantiate Services
  • Application-Services (classes from app\service) MUST be injected with the @Inject annotation
  • Controllers SHOULD use ServiceInterfaces and NOT concrete implementations

Example usage:

<?php

namespace app\controller;

use app\service\AccountServiceInterface;

class AccountController{

    /**
     * @var \app\service\AccountServiceInterface
     */
    private $service;


    /**
     * @param $service app\service\AccountServiceInterface
     * @Inject("service", "app\service\AccountService")
     */
    public function __construct(AccountServiceInterface $service){
        $this->service = $service;
    }

}

2. User input¶

  • Models SHOULD NOT be instantiatet in a controller
  • User input SHOULD NOT be applied to a model from a controller method
  • Models SHOULD be passed as a parameter to a controller method
  • The User input should by automatically applied by the framework to a model

Wrong usage:

<?php

namespace app\controller;

use app\service\AccountServiceInterface;
use Symfony\Component\HttpFoundation\Request;
use app\model\Account;
use Symfony\Component\HttpFoundation\RedirectResponse;

class AccountController{

    /**
     * @var \app\service\AccountServiceInterface
     */
    private $service;


    /**
     * @param $service app\service\AccountServiceInterface
     * @Inject("service", "app\service\AccountService")
     */
    public function __construct(AccountServiceInterface $service){
        $this->service = $service;
    }

    public function create(Request $request){
        $account = new Account();
        $account->setName($request->request->get('name'))
                ->setMail($request->request->get('mail'));
        // ....
        $this->service->create($account);
        return new RedirectResponse('/accounts/' . $account->getId() . '/');
    }

}

Right usage:

<?php

namespace app\controller;

use app\service\AccountServiceInterface;
use app\model\Account;
use Symfony\Component\HttpFoundation\RedirectResponse;

class AccountController{

    /**
     * @var \app\service\AccountServiceInterface
     */
    private $service;


    /**
     * @param $service app\service\AccountServiceInterface
     * @Inject("service", "app\service\AccountService")
     */
    public function __construct(AccountServiceInterface $service){
        $this->service = $service;
    }

    public function create(Account $account){
        $this->service->create($account);
        return new RedirectResponse('/accounts/' . $account->getId() . '/');
    }

}

Note

You have to specify the accessible properties in the file: config/metadata/property-access.php

3. Validation¶

  • MUST NOT be done in a controller!
  • SHOULD be done in a service
  • The provided validator MUST be used

Note

See http://symfony.com/doc/current/book/validation.html
Configuration file of the Framework: config/validation/validation.xml
